h1. AI赋能爱快路由器测试效率提升方案

----

h2. 背景与目标

爱快路由器4.0测试团队已建立一套基于Playwright的Python UI自动化测试框架，覆盖VLAN、IP限速、MAC限速等模块。在此基础上，引入Claude Code + MCP工具链，目标是：

* *提升测试编写效率*：AI辅助生成测试用例和自动化代码
* *增加验证深度*：从纯UI验证扩展到数据库→iptables→内核→实际带宽的全链路验证
* *降低人工成本*：AI执行冒烟测试、自动总结BUG、格式化报告
* *持久化测试知识*：通过AI记忆系统沉淀测试经验和模式

----

h3. 方向4：SSH后台深度数据验证

*方法：* Claude Code + MCP-SSH登录路由器后台，多层次验证前端操作结果

*5层验证体系（由浅入深）：*

||层级||验证内容||命令||置信度||
|L1 数据库|配置数据是否写入|/usr/ikuai/function/simple_qos show|必要不充分|
|L2 iptables|限速规则是否下发|iptables -L IP_QOS -n -v|必要不充分|
|L3 ipset|目标IP是否加入集合|ipset list _simple_qos_ID|必要不充分|
|L4 内核|ik_core模块 + dmesg|lsmod / dmesg|辅助判断|
|L5 iperf3|实际带宽测量|iperf3 -c server -B src_ip -J|*充分证明*|

*关键洞察：* 数据库有数据 ≠ 功能正常。必须验证iptables规则实际下发到IP_QOS链。

h3. 方向5：全链路测试实战

完整的全链路验证流程（以IP限速为例）：

{code}
步骤1: 前端配置（MCP-Playwright / Python自动化）
  → 打开终端限速页面，新增规则：IP=192.168.148.2, 上传=2048KB/s, 下载=4096KB/s

步骤2: L1 数据库验证（MCP-SSH / Paramiko）
  → /usr/ikuai/function/simple_qos show | jq '.data[] | select(.tagname=="全链路测试")'
  → 验证 upload=2048, download=4096

步骤3: L2 iptables规则验证
  → iptables -L IP_QOS -n -v
  → 确认存在 "limit: 2048 kBps mode srcip" (上传) 和 "limit: 4096 kBps mode dstip" (下载)

步骤4: L3 ipset验证
  → ipset list _simple_qos_{id}
  → 确认Members包含 192.168.148.2

步骤5: L4 内核验证
  → lsmod | grep ik_core → 确认模块加载
  → dmesg | tail -50 | grep -iE "error|fail" → 确认无异常

步骤6: L5 iperf3实测（可选，较耗时）
  → 添加策略路由让流量经过路由器
  → iperf3 -c 10.66.0.40 -B 192.168.148.2 -t 10 -J
  → 对比实测带宽与配置值（允许20%误差）
{code}

*实测结果（2026-03-04确认）：*
* 上传：实测18.66 Mbps vs 配置16.38 Mbps (2048 KB/s × 8) → +14%，限速生效
* 下载：实测36.38 Mbps vs 配置32.77 Mbps (4096 KB/s × 8) → +11%，限速生效
* 大量TCP retransmit证实限速通过丢包机制实现

h2. iKuai QoS架构发现

h3. 核心发现（2026-03-04实测）

iKuai路由器 *不使用* 标准Linux QoS工具（tc/htb/tbf），而是通过自定义方案实现：

{code}
自定义内核模块 ik_core (1.4MB)
    │
    ├── iptables filter表 FORWARD链
    │   └── IP_QOS / MAC_QOS 自定义链
    │       └── 自定义target: "limit: XXXX kBps mode dstip/srcip"
    │
    └── ipset 管理限速目标
        └── simple_qos_{id} (list:set) → _simple_qos_{id} (hash:net)
{code}

h3. FORWARD链调用顺序

{code}
FORWARD → AUTH_EXPIRES → ACL_MAC → FLOW_CONTROL → WECOM_BLOCK_MAC
        → CONNLIMIT → LAYER7 → QOS → SIMPLE → IP_QOS → MAC_QOS
        → COUPON_QOS → WEBAUTH_QOS → OVPN_QOS → L2TP_QOS → PPTP_QOS
        → PPPOE_QOS → IKE_QOS → FIREWALL → LAN_VISIT → NONAT
        → WEBAUTH → PPPOE → PPP → ARP
{code}

h3. 对测试的影响

* 不能用{{tc class show}}或{{iptables -t mangle}}查规则，要看{{iptables -L IP_QOS -n -v}}（filter表）
* ipset是两层嵌套结构，外层list:set引用内层hash:net
* 限速通过丢包实现，所以iperf3会有大量retransmit

h3. BackendVerifier API

{code:python}
from utils.backend_verifier import BackendVerifier

# 创建验证器
verifier = BackendVerifier()
verifier.connect_router()

# L1: 数据库验证
result = verifier.verify_qos_database("simple_qos", tagname="规则名")

# L2: iptables验证
result = verifier.verify_iptables_rule("IP_QOS", rule_id=1, expected_speed_kbps=2048)

# L3: ipset验证
result = verifier.verify_ipset_member(rule_id=1, expected_ip="192.168.148.2")

# L4: 内核验证
result = verifier.verify_kernel()

# L5: iperf3实测
result = verifier.verify_iperf3("upload", expected_kbps=2048)

# 全链路一键验证
chain_result = verifier.verify_ip_qos_full_chain(
    tagname="规则名", ip="192.168.148.2",
    upload_kbps=2048, download_kbps=4096,
    run_iperf3=False  # True则包含L5
)
print(chain_result.summary())

verifier.close()
{code}
